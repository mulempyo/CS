
운영체제 핵심 개념 정리 (설명 포함)

실시간 시스템: 정해진 시간안에 입력에 대한 정확한 출력을 보장하는 시스템, 크고 작은 사고 방지와 높은 효율을 위해 사용
printf같은 로그함수는 실시간성 깨트림, 페이지폴트 고려와 프로세스 스케줄링 기반으로 실시간성 가지게 해야함.
RCL, RMW 사용자정의 메모리 할당자, 페이지 폴트 최소화
노드(계산 수행하는 하나의 프로세스) 
노들릿(단일 프로세스에서 동작되는 복수개의 노드들을 위한 제로카피 제공하여 퍼블리셔와 서브스크라이버간 데이터 통신에서 고정된 메모리 공간 사용하도록 함)

CPU 구성: ALU, 제어장치, 레지스터
- 프로그램 카운터(PC): 다음 실행 명령어 주소 저장
- 명령어 레지스터(IR): 현재 실행 중인 명령어 저장
- 버퍼(Buffer): 속도 차이를 완화하는 임시 저장 공간
- 캐시(Cache): CPU와 RAM 사이의 고속 메모리
- 인터럽트: CPU 작업을 중단시키고 특정 작업 처리
- DMA: CPU를 거치지 않는 메모리 접근
- 병렬처리: 여러 작업을 동시에 수행
- 저장장치 계층구조: cpu → 캐시 → 메인메모리 → SSD → HDD
- 휘발성 메모리: RAM, Cache
- 비휘발성 메모리: SSD, HDD, ROM, 플래시 메모리

2. 프로세스와 스레드

폰노이만 구조: 프로그램이 메모리에 올라와서 작업이 진행된다.

- 프로세스(Process)
  실행 중인 프로그램. 독립적인 메모리 공간(Code, Data, Heap, Stack)을 가진다.

- PCB(Process Control Block)
  운영체제가 프로세스를 관리하기 위해 사용하는 자료구조.
  PID, 상태, 레지스터 값, 프로그램 카운터 등의 정보가 저장된다.

- 문맥교환(Context Switch)
  CPU가 한 프로세스의 실행 상태를 저장하고 다른 프로세스의 상태를 복원하는 과정.
  cpu를 차지하던 프로세스가 나가고 새로운 프로세스를 받아들인다.

- 프로세스 상태
  New(생성) → Ready(준비) → Running(실행)
  → Waiting(대기) → Terminated(완료)
ROS2의 Lifecycle
create, configure, cleanup, activate, deactivate, shutdown, destroy
- 프로세스 구조: 정적할당영역(코드, 데이터), 동적할당영역(힙, 스택)
  Code: 실행 코드
  Data: 전역변수, static 변수, 데이터
  Heap: 동적 할당 메모리
  Stack: 함수 호출 정보, 지역변수

- 좀비 프로세스
  실행 종료되었지만 부모 프로세스가 종료 상태를 회수하지 않은 프로세스.

- 스레드(Thread)
  프로세스 내부의 실행 단위.

- 멀티스레드
  하나의 프로세스에서 여러 스레드를 동시에 실행하는 방식.

- 멀티태스킹
  CPU가 여러 작업에 사용시간을 잘게 쪼개 배분하여 동시에 실행되는 것처럼 보이게 하는 기술.

- 멀티프로세싱
  여러 CPU 또는 여러 코어를 이용해 실제 병렬 수행하는 방식.

- CPU 멀티스레드
  하나의 코어가 여러 스레드를 처리하는 기술(Hyper-Threading).

- 멀티스레드 장점
  응답성, 효율성 향상, 자원 공유, CPU 활용도 증가

- 멀티스레드 단점
  동기화 문제, 경쟁 상태

- 프로세스와 스레드 차이
  프로세스는 메모리를 독립적으로 사용.
  스레드는 같은 프로세스 내 메모리를 공유.

- read()
  파일이나 장치에서 데이터를 읽는 시스템 콜.

- write()
  파일이나 장치에 데이터를 쓰는 시스템 콜.

- fork()
  현재 프로세스를 복제하여 자식 프로세스를 생성.

- exec()
  현재 프로세스를 새로운 프로그램으로 교체.

3. CPU 스케줄링: 어떤 작업에 cpu 사용시간을 배정할지 결정, cpu사용률, 처리량, 대기시간, 응답시간, 반환시간 기준으로 스케줄링 알고리즘 선택함.

- 스케줄링 목적
  공정성, 효율성, 안정성, 확장성, 반응시간보장, 무한연기방지

- 선점형 스케줄링
  운영체제가 강제로 CPU를 회수 가능.
  시분할 방식에 사용하고 문맥교환 오버헤드가 많다.

- 비선점형 스케줄링
  작업이 끝나기 전까지 CPU를 회수하지 않음.
  일괄작업방식에 사용하고 대기 프로세스가 많아 처리율이 떨어짐

- CPU 집중 프로세스
  계산이 많은 프로세스.

- 입출력 집중 프로세스
  I/O 작업 비중이 높은 프로세스.

- FCFS
  먼저 도착한 작업을 먼저 실행.

- SJF
  실행 시간이 가장 짧은 작업을 우선 실행.

- HRN
  아사 현상을 해결하기 위해 대기시간과 cpu 사용시간을 고려해 우선순위를 결정하는 알고리즘
    
- RR(Round Robin)
  일정 시간마다 CPU를 교체(작업을 다 못끝내면 준비 큐 맨뒤로 가서 대기), 우선순위 적용 안됨

- SRT
  라운드 로빈 + 준비큐에서 잔류시간이 가장 적은 프로세스 우선 선택

- 다중 큐
  프로세스를 여러 큐로 나누어 관리, 우선순위에 따라 준비 큐 여러개 사용

- 다중 피드백 큐  
  cpu 사용 후 프로세스가 우선순위가 하나 원래 큐가 아닌 낮은 큐로 들어감.

- Priority
  우선순위가 높은 작업을 먼저 실행.(선점 비선점 둘다됨, 우선순위 구현은 정해져있지 않다.)


4. 동기화: 여러 스레드나 프로세스가 공유 자원에 동시에 접근할 때 데이터 충돌이 발생하지 않도록 실행 순서를 조정하는 기법
   동기화 일반적 뜻: 여러 개의 독립적인 것들의 동작 시점이나 순서를 맞추는 것

- 통신 방향
  단방향(Simplex), 반이중(Half Duplex), 전이중(Full Duplex)

- 프로세스 내부통신(전역변수, 파이프), 프로세스 간 통신(공용파일, 파이프), 네트워크 통신(소켓)  

- 바쁜 대기(Busy Waiting)
  상태변화를 알기 위해 반복문을 무한 실행하며 대기하는 것

- 공유 자원
  여러 프로세스가 함께 사용하는 자원(변수, 메모리, 파일 등).

- 경쟁 상태(Race Condition)
  2개 이상 프로세스가 공유 자원을 병행적으로 읽거나 쓸려 하는 상황. 경쟁상태에선 실행 순서에 따라 결과가 달라질수있다.

- 임계구역(Critical Section)
  공유 자원에 접근하는 코드 영역. 공유 자원 접근 순서에 따라 결과가 달라지는 프로그램 영역

- 생산자 소비자 문제
  생산자와 소비자가 버퍼를 공유할 때 발생하는 동기화 문제.

- 임계구역 해결 조건: 해결 방법은 잠금을 이용하는것
  상호배제, 진행의 윤통성, 한정대기

- 세마포어
  정수 변수를 이용한 동기화 기법, p: 1감소 시키고 잠금, v: 1 증가시키고 잠금 해제, 공유 자원 여러 개 일때 여러 잡업이 동시에 접근가능

- 뮤텍스
  lock/unlock 기반 상호배제 기법.

- 모니터
  세마포어를 추상화한(인터페이스만 제공) 고수준 동기화 기법. 세마포어의 p,v연산 없이 자동처리

5. 교착상태

- 교착상태(Deadlock)
  서로 자원을 기다리며 무한 대기하는 상태.

- 필요조건 4가지
  상호배제, 비선점, 점유와 대기, 원형대기

- 아사현상(Starvation)
  특정 프로세스가 계속 자원을 받지 못하는 상태.(작업 무한 연기)

- 해결 방법
  예방(4가지 조건 무력화), 회피, 검출(자원할당 그래프 이용 발견), 회복(검출 후 해결)

- 은행원 알고리즘
  안전 상태(기대 자원과 비교했을때 가용자원이 크거나 같은경우가 한번 이상인 경우)에서만 자원을 할당하는 회피 알고리즘.

6. 메모리 관리

- 물리 메모리
  실제 RAM 공간.

- MMU(메모리 관리 장치)
  가상주소를 물리주소로 변환하는 장치.

- 상대주소
  프로세스 기준 주소.

- 스왑
  프로세스를 디스크와 메모리 사이에서 이동.

- 스왑인
  디스크 → 메모리

- 스왑아웃
  메모리 → 디스크

- 고정분할
  미리 정해진 크기로 메모리 분할.

- 가변분할
  필요한 크기만큼 메모리 분할.

- 내부단편화(고정 분할 단편화)
  할당된 공간 내부 낭비.

- 외부단편화(가변분활 단편화)
  사용 가능한 공간이 흩어져 있는 현상.

7. 가상 메모리

- 가상 메모리
  실제 RAM크기와 상관없이 메모리를 이용할 수 있도록 하는 기술.

- 페이지
  가상 주소의 분할된 각 영역  

- 페이징
  고정분할방식의 가상메모리 관리기법.

- 세그먼테이션
  가변분할방식의 가상메모리 관리기법.

- 세그먼테이션 페이징 혼용기법
  가상메모리를 먼저 세그먼트로 나눈 뒤, 각 세그먼트를 다시 페이지 단위로 분할해 물리 메모리에 배치하는 방식 

8. 페이지 관리

- 지역성(Locality)
  최근성 또는 인접성

- 무작위(간단)
  무작위로 페이지 스왑영역으로 보냄

- FIFO(간단)
  가장 먼저 들어온 페이지 스왑영역으로 보냄.

- 최적(이론적)
  미래의 접근 패턴 기반으로 페이지 선정 후 스왑영역으로 보냄

- LRU(최적 근접)
  가장 오래 사용하지 않은 페이지 스왑영역으로 보냄

- LFU(최적 근접)
  가장 적게 사용된 페이지 스왑영역으로 보냄

- NUR(최적 근접)
  최근에 사용한적 없는 페이지 스왑영역으로 보냄

- FIFO 변형(최적 근접)
  FIFO를 변형하여 성능 높이기  

- 스레싱
  페이지 부재가 과도하게 발생하여 성능이 저하되는 현상.

- 페이지 부재(Page Fault)
  필요한 페이지가 물리메모리(RAM)에 없는 상태.

9. 보호 모드 및 메모리 구조

- 사용자 모드
  제한된 권한으로 실행.

- 커널 모드
