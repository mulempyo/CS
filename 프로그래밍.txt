1. A* 
그래프 탐색 기반 최단 경로 탐색 알고리즘

시작점에서 목표점까지의 경로를 탐색
실제 이동 비용 g(n)과 이웃노드에서 목표까지의 추정 비용 h(n)을 함께 사용
평가 함수
f(n)=g(n)+h(n)
최단 경로를 찾을 수 있음
현재까지의 비용과 목표까지의 예상 비용을 이용하여 최적 경로를 찾는 그래프 탐색 알고리즘입니다.

ros::NodeHandle nh;이 생성자에 없다면 initialize()에서 토픽 발행, 파라미터 읽기 등을 하면 정상 초기화가 안 될 수 있음
스코프 락은 예외가 발생해도 mutex가 해제되지 않는 문제를 방지때문에 사용
플래너에서 뮤텍스 사용은 move_base에서 플래너에서 코스트맵을 읽을때 맵 초기화 콜백함수에서 초기화 시키지 않도록 하기 위해서 쓴다.
unsigned int를 써서 2차원 좌표를 1차원으로 만드는 이유는 2차원 배열보다 1차원 배열이 접근 비용이 덜 들어서이다.
A*에서 우선순위 큐를 사용하는 가장 큰 이유는 현재 탐색해야 할 노드 중 우선순위(f값)가 가장 높은(또는 비용이 가장 낮은) 노드를 빠르게 꺼내기 위해서입니다.

potentialFieldCost: 현재 셀 주변의 장애물들을 조사하여 장애물에 가까울수록 큰 비용(반발 비용)을 계산하는 함수

2. RRT 
랜덤 샘플링으로 트리를 확장하여 경로를 찾는 무작위 샘플링 기반 경로 탐색 알고리즘

공간에 랜덤 점을 생성
기존 트리에서 가장 가까운 노드를 찾음
일정 거리만큼 확장
목표에 도달할 때까지 반복

특징
복잡한 공간 탐색에 강함
빠르게 경로 생성 가능
최적 경로 보장은 없음
목적지에 잘 도착 못할수도있다.

3. RRT* 
RRT에 재배선(Rewiring)을 추가하여 경로 비용을 지속적으로 개선하는 최적 경로 탐색 알고리즘
RRT에 추가로
새 노드 생성 후 주변 노드 탐색
부모 재선택(Rewire)
경로 비용이 더 작아지도록 트리 구조 수정

특징
시간이 지날수록 경로 품질 향상
점근적 최적성(Asymptotic Optimality)

4. IRRT*
RRT*를 더 빠르게 최적화한 알고리즘

최초 경로를 찾은 뒤, 전체 공간을 탐색하지 않음
시작점과 목표점을 초점으로 하는 타원 내부만 샘플링

특징
불필요한 샘플 감소
최적 경로 수렴 속도 향상

visualization_msgs::Marker::LINE_LIST: 점 2개를 한 쌍으로 묶어서 선분을 그리겠다.
visualization_msgs::Marker::ADD: Marker를 새로 추가하거나 갱신하겠다.
visualization_msgs::Marker::LINE_STRIP: 점들을 순서대로 연결해서 하나의 선으로 그린다.
visualization_msgs::Marker::SPHERE: 특정 위치에 구(원)를 하나 그린다.

단순히 상대이동량을 더하면 진짜 로봇의 포즈를 파악하는것이 불가능하다.
포즈(위치 + 방향)를 고려해서 상대이동량을 고려해야 정확한 이동과 포즈 파악이 된다.
예: 위치 : (0,0)
방향 : 90도 일때
그냥더하기: (0,0) -----> (1,0)
포즈고려: new_x = 0 + 0*1 - 1*0 = 0
new_y = 0 + 1*1 + 0*0 = 1
(0,0) -----> (0,1)

공간적 일관성 검사의 한계: 라이다 특성상 다음의 어느 포인트와 대응되는지 파악하는것이 어렵다는것이 한계

micro-ROS와의 UART 통신은 DMA 기반 전송방식을 사용하여 CPU 개입을 최소화하고 데이터 끊김 문제를 방지

SystemClock_Config(): MCU 전체 동작 속도를 정하는 부분.
MX_TIM2_Init(), MX_TIM4_Init(): 엔코더 값을 읽기 위한 타이머 설정.
MX_TIM3_Init(): PWM 출력용 타이머 설정.
MX_USART2_UART_Init(): UART 통신 설정.
MX_DMA_Init(): DMA와 인터럽트 설정.
MX_GPIO_Init(): 핀 입출력 설정.

UART: MCU와 다른 장치가 직렬 통신(serial communication)하는 방식, STM32와 Jetson Nano가 선 2개(TX, RX)를 이용해서 데이터를 주고받는 것
I2C: SDA와 SCL 두 선을 이용해 여러 센서를 연결하는 저속 직렬 통신
SPI: MOSI, MISO, SCK, CS사용하고 I2C보다 더 빠르고 전이중 통신이 가능하지만 선이 더 많이 필요.
CAN: 여러 노드가 하나의 버스를 공유하며 통신하는 방식.
SDA, SCL, TX(송신), RX(수신), MOSI, MISO: 핀의 역할, 핀의 이름이 아니다.
FreeRTOS: MCU에서 사용할 수 있는 실시간 운영체제(일반적인 Linux나 Windows보다 훨씬 가볍지만 Task 스케줄링, 동기화, Queue 등의 운영체제 기능을 제공)
